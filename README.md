# ARIA --- 動態災害風險分析與避難所決策支援系統

本專案以颱風期間降雨資料為基礎，結合地形風險與空間分析，建立一套「動態風險疊合（Dynamic
Risk Overlay）」系統，並透過互動式地圖與生成式
AI，提供避難決策支援。系統以宜蘭與花蓮地區為主要分析範圍，模擬颱風極端降雨情境。

------------------------------------------------------------------------

## 📊 系統架構

本系統整合三大模組：

1.  **雨量資料處理（CWA / CoLife）**
    -   即時 API / 模擬資料統一格式處理\
    -   缺值處理（-998 → NaN）
2.  **空間分析（GeoPandas）**
    -   CRS 統一（EPSG:3826）\
    -   buffer（5km）影響範圍分析\
    -   空間連接（sjoin）
3.  **決策支援與視覺化（Folium + Gemini AI）**
    -   風險分級地圖\
    -   AI 即時應變建議

------------------------------------------------------------------------

## 🔹 核心功能

### A. LIVE / SIMULATION 模式切換

-   LIVE：使用中央氣象署 API\
-   SIMULATION：使用 JSON（颱風峰值）\
-   使用 normalize function 統一資料格式

------------------------------------------------------------------------

### B. 動態風險疊合（Dynamic Risk Overlay）

風險分級邏輯：

-   **CRITICAL**：時雨量 \> 80mm\
-   **URGENT**：時雨量 \> 40mm 且 terrain_risk = HIGH\
-   **WARNING**：時雨量 \> 40mm 或 terrain_risk = HIGH\
-   **SAFE**：其餘

分析流程：

1.  雨量站轉換 CRS（4326 → 3826）\
2.  建立 buffer（5km）\
3.  與避難所做 sjoin\
4.  找最近雨量站\
5.  計算動態風險

------------------------------------------------------------------------

### C. 視覺化（Folium）

地圖包含：

-   宜蘭＋花蓮鄉鎮市界\
-   雨量站（CircleMarker）\
-   雨量 HeatMap\
-   避難所（顏色代表風險）\
-   Popup 顯示詳細資訊

------------------------------------------------------------------------

### D. Bonus：Gemini AI 戰術顧問

-   選取 Top 3 高風險避難所\
-   輸入：
    -   雨量
    -   地形風險
    -   最近測站
    -   風險等級
-   輸出：
    -   3 條具體應變建議
-   顯示：
    -   Folium Popup

## 🧠 AI 診斷日誌（Debug Log）

### 1️⃣ CRS 未對齊（sjoin 為空）

**問題：** rain_gdf shape: (0, 6)

**原因：** 資料 CRS 不一致（4326 vs 3826）

**解法：** rain_gdf = rain_gdf.to_crs(epsg=3826)

**結論：** 👉 空間分析前必須統一 CRS

------------------------------------------------------------------------

### 2️⃣ CWA -998 缺值問題

**問題：** HeatMap 顏色異常

**原因：** -998 表示 missing

**解法：** df\["rain_1hr"\] = df\["rain_1hr"\].replace(-998, np.nan)

------------------------------------------------------------------------

### 3️⃣ 經緯度順序錯誤（lat/lon）

**問題：** 點位跑到海上

**原因：** Folium 使用 \[lat, lon\]

**解法：** lat = row.geometry.y\
lon = row.geometry.x

------------------------------------------------------------------------

### 4️⃣ 避難所點位錯誤（資料錯位）

**問題：** 澳花國小出現在新竹

**原因：** 資料 join 未檢查

**解法：** - 使用縣市篩選\
- 經緯度合理性檢查

------------------------------------------------------------------------

### 5️⃣ HeatMap 山區盲區

**問題：** 山區無資料

**原因：** 測站集中平地

**改善方向：** - IDW\
- Kriging\
- 雷達降雨
